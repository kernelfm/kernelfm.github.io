    • %keys% - Количество ключей игрока именно для этого кейса.
    • %all_keys% - Общее количество ключей игрока по всем кейсам.
    • %case_display_name% - Отображаемое имя кейса.
    • %username% - Ник игрока.

    Режимы действий при выигрыше наград:

    • [command] <команда> - Выполняет команду от консоли. Поддерживает плейсхолдеры.
    • [broadcast] <сообщение> - Отправляет сообщение всем игрокам на сервере (поддерживает HEX и цвета).
    • [message] <сообщение> - Отправляет личное сообщение игроку, который открыл кейс.

    Разработка аддонов для плагина (создание своих анимаций)

    LostCases спроектирован так, что разработчики могут легко добавлять новые анимации с помощью внешних Jar-файлов (аддонов), загружаемых динамически из папки addons/.

    Шаги по созданию аддона:

    1. Создайте новый Java-проект с зависимостью от Spigot/Paper API и от скомпилированного Jar-файла плагина LostCases.
    2. Создайте главный класс аддона, который имплементирует интерфейс ru.lostdev.lostCases.api.CaseAddon.
    3. В методе onLoad(LostCases plugin) зарегистрируйте вашу анимацию с помощью:
      plugin.getAnimationManager().registerAnimation(new MyCustomAnimation(plugin));
    4. Скомпилируйте проект в Jar-архив и поместите его в папку plugins/LostCases/addons/.

    Описание интерфейсов API:

    • CaseAddon - Точка входа для Jar-файла аддона. Содержит методы onLoad(LostCases plugin) и onUnload() для инициализации и освобождения ресурсов аддона.
    • CaseAnimation - Класс, представляющий тип анимации. Должен возвращать имя анимации через getName() (например, "chests") и возвращать сессию открытия через createSession(...).
    • AnimationSession - Класс, управляющий конкретным открытием кейса игроком в мире. Содержит методы:
      • start() - Вызывается при запуске анимации. Здесь вы расставляете блоки анимации и запускаете задачи (Scheduler).
      • handleInteract(Player, Block, Action) - Вызывается при кликах игрока по блокам во время открытого кейса. Позволяет делать выбор сундука.
      • stop() - Вызывается при завершении или отмене сессии. Обязательно восстановите все измененные блоки, удалите голограммы и вызовите plugin.getAnimationManager().endSession(player, caseBlockLoc).

    Пример простейшей анимации (Java-код):

    package myaddon.animation;

import org.bukkit.Location;
import org.bukkit.block.Block;
import org.bukkit.configuration.ConfigurationSection;
import org.bukkit.entity.Player;
import org.bukkit.event.block.Action;
import ru.lostdev.lostCases.LostCases;
import ru.lostdev.lostCases.api.AnimationSession;
import ru.lostdev.lostCases.api.CaseAnimation;
import ru.lostdev.lostCases.case_system.CaseModel;

public class MyCustomAnimation implements CaseAnimation {
    private final LostCases plugin;

    public MyCustomAnimation(LostCases plugin) {
        this.plugin = plugin;
    }

    @Override
    public String getName() {
        return "my_custom_anim";
    }

    @Override
    public AnimationSession createSession(Player player, Location location, CaseModel caseModel, ConfigurationSection config) {
        return new AnimationSession() {
            @Override
            public void start() {
                player.sendMessage("Анимация запущена!");
                // Логика отображения эффектов
            }

            @Override
            public void handleInteract(Player p, Block block, Action action) {
                // Выбор сундука или предмета
            }

            @Override
            public void stop() {
                plugin.getAnimationManager().endSession(player, location);
            }
        };
    }
}
